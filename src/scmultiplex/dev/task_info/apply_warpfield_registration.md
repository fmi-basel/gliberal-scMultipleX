## Purpose

- Apply precomputed warpfield registration (generated by `calculate_warpfield_registration`) to all channels of a moving OME-Zarr image.
- Create a new registered OME-Zarr image with the user-defined suffix (e.g. `_registered`).
- Warp each ROI independently using the corresponding on-disk warp map (`registration/<registration_name>/<label>.npz`).
- For the reference round, create a registered copy containing only the ROIs present in the selected ROI table.
- Optionally mask each registered ROI by its parent label after registration to prevent overlapping regions.
- Optionally copy selected folders, ROI tables, and corresponding label images to the registered outputs.

## Inputs

- **roi_table_name**: Name of the generic ROI table used to load corresponding regions in the reference and moving images. Must match the ROI table used during warpfield calculation.
- **registration_name** *(default: `warpfield`)*: Name of the registration subfolder containing the saved warp maps.
- **output_image_suffix** *(default: `registered`)*: Suffix appended to the output registered image name.
- **mask_output_by_parent** *(default: `False`)*: If `True`, masks each registered ROI by its parent label before writing to the output image.
- **folders_to_copy_from_unregistered_zarr** *(optional)*: List of folders (e.g. `meshes`, `plots`, `registration`) to copy from the original image to the registered image.
- **tables_to_propagate_from_ref_to_mov** *(optional)*: List of ROI tables to copy from the reference image to each registered moving image. Corresponding label images are copied automatically for masking ROI tables.
- **overwrite_reference** *(default: `False`)*: Overwrite the registered reference image if it already exists.
- **overwrite_moving** *(default: `False`)*: Overwrite registered moving images if they already exist.


## Outputs

- A new registered OME-Zarr image for each processed acquisition, named with the specified `output_image_suffix`.
- Registered image data for all channels, written ROI-by-ROI.
- Optionally copies folders from the original image (e.g. `meshes`, `plots`, `registration`).
- Optionally propagates specific ROI tables and their corresponding label images from the reference round to moving rounds.
- Updates well metadata and Fractal image list to include the newly created registered image.

## Limitations

- Requires warp maps generated previously by `calculate_warpfield_registration`; ROIs without a corresponding warp map are skipped with a warning.
- Registration is applied independently to each ROI and does not enforce continuity across ROI boundaries.
- ROI IDs and ROI geometry must correspond to those used when generating the warp maps. Each ROI must match in label ID and shape between moving and reference round.
- Requires the `warpfield` package and compatible GPU runtime.
